Common functions in MATLAB about sparse matrix

Run

>> help sparfun

or click here.

Create sparse matrix

sparse

  • S = sparse(A) converts a full matrix into sparse form by squeezing out any zero elements. If a matrix contains many zeros, converting the matrix to sparse storage saves memory.

  • S = sparse(m,n) generates an m-by-n all zero sparse matrix.

>> A = eye(10000);
>> SA = sparse(A);
>> whos A SA

  Name          Size                   Bytes  Class     Attributes

  A         10000x10000            800000000  double              
  SA        10000x10000               240008  double    sparse

sprand, sprandsym

  • R = sprandn(S) creates a sparse matrix that has the same sparsity pattern as the matrix S, but with normally distributed random entries with mean 0 and variance 1.

  • R = sprandn(m,n,density) creates a random m-by-n sparse matrix with approximately density*m*n normally distributed nonzero entries for density in the interval [0,1].

>> S = sprandn(5,5,0.5);

   (1,2)       1.7036
   (3,2)      -1.5975
   (3,3)      -0.8973
   (4,3)      -1.7799
   (5,3)       0.5464
   (1,4)       1.9679
   (1,5)      -0.9504
   (2,5)      -0.0473
   (3,5)      -0.0197
   (5,5)      -0.3547

  • R = sprandsym(S) returns a symmetric random matrix whose lower triangle and diagonal have the same structure as S. Its elements are normally distributed, with mean 0 and variance 1.

  • R = sprandsym(n,density) returns a symmetric random, n-by-n, sparse matrix with approximately density*n*n nonzeros; each entry is the sum of one or more normally distributed random samples, and (0 <= density <= 1).

%%

S = sprandn(10,10,0.5); 
subplot(1,3,1); spy(S); title('S');
SSl = sprandsym(S); 
subplot(1,3,2); spy(SSl); title('SSl');
SSu = sprandsym(S');
subplot(1,3,3); spy(SSu); title('SSu');

speye

  • S = speye returns a sparse scalar 1.

  • S = speye(n) returns a sparse n-by-n identity matrix, with ones on the main diagonal and zeros elsewhere.

  • S = speye(n,m) returns a sparse n-by-m matrix, with ones on the main diagonal and zeros elsewhere.

%% speye

S1 = speye(100); subplot(1,2,1); spy(S1);
S2 = speye(100,150); subplot(1,2,2); spy(S2);

Display sparse matrix

spy

  • spy(S) plots the sparsity pattern of matrix S. Nonzero values are colored while zero values are white. The plot displays the number of nonzeros in the matrix, nz = nnz(S).

  • spy(S,LineSpec) additionally specifies LineSpec to give the marker symbol and color to use in the plot.

%% spy

S = bucky;
subplot(1,2,1); spy(S);
subplot(1,2,2); spy(S, 'r*')

full

A = full(S) converts sparse matrix S to full storage organization, such that issparse(A) returns logical 0 (false).

%% full

S = speye(5);
A = full(S);
disp(issparse(A))
disp(A)

Conjugate Gradients

Algorithm

Let A(N×N)A (N\times N)A(N×N) be symmetric positive definite (SPD), that is AT=AA^T=AAT=A and all eigenvalues is positive, or say xTAx>0x^TAx>0xTAx>0 for any x≠0x\neq \bf 0x=0. Let bbb be an N×1N\times 1N×1 vector, we want to solve a x∗x_*x∗​ s.t. Ax∗=bAx_*=bAx∗​=b, since AAA is SPD AAA is non singular, and hence x∗x_*x∗​ is existed and unique.

Conjugate Gradients (CG) find x∗x_*x∗​ iteratively starting from x0x_0x0​, x0x_0x0​ is could be any vector, and we will use 0\bf 00. And xnx_nxn​ belongs to the Krylov space:

Kn=⟨b,Ab,⋯ ,An−1b⟩K_n=\lang b, Ab,\cdots,A^{n-1}b\rangKn​=⟨b,Ab,⋯,An−1b⟩

which is spanned by vectors b,Ab,⋯ ,An−1bb, Ab,\cdots,A^{n-1}bb,Ab,⋯,An−1b, and any k∈Knk\in K_nk∈Kn​ is a linear combination of these vectors. It is direct to see that Kn⊆RNK_n\subseteq \mathbb{R}^NKn​⊆RN, so conjugate gradient is searching in bigger and bigger space for a good approximation to the solution.

Let define a special way to measure the size of a vector: define "AAA - norm" of a vector xxx as

∥x∥A=xTAx\|x\|_A=\sqrt{x^TAx}∥x∥A​=xTAx​

So ∥x∥A\|x\|_A∥x∥A​ is a positive real number if x≠0x\neq \bf 0x=0, because by definition, AAA is symmetric positive definite, and therefore xTAxx^TAxxTAx is a positive number.

And define the error of each steps: define nnn-th error as

en=x∗−xne_n=x_*-x_nen​=x∗​−xn​

and define nnn-residual as

rn=Aen=b−Axnr_n=Ae_n=b-Ax_nrn​=Aen​=b−Axn​

In principle, we like a small error, but in practice, the thing we can measure is the residual. Thus we minimize the residual as a mean of minimizing.

THM 1. Among all x∈Knx\in K_nx∈Kn​, there exists one unique xnx_nxn​ such that minimizes ∥en∥A\|e_n\|_A∥en​∥A​.

So at every step of conjugate gradient, we can find the best approximation in a certain measure. And from this theorem, we obtain a corollary: ∥e0∥A≥∥e1∥A≥⋯≥0\|e_0\|_A\geq \|e_1\|_A\geq \cdots \geq 0∥e0​∥A​≥∥e1​∥A​≥⋯≥0. That is the error keep going down monotonically.

Define f(x)=12xTAx−xTbf(x)=\frac{1}{2}x^{T}Ax-x^{T}bf(x)=21​xTAx−xTb, thus ∇f(x)=Ax−b\nabla f(x)=Ax-b∇f(x)=Ax−b, now we want Ax=bAx=bAx=b, which means we want ∇f(x)=0\nabla f(x)=0∇f(x)=0, i.e minimize of f(x)f(x)f(x).

Now we can introduce the conjugate gradient algorithm. First, we start form an initial guess x0=0x_0=\bf 0x0​=0, corresponding to this is a residual r0=b−Ax0=b−A0=br_0=b-Ax_0=b-A\mathbf{0}=br0​=b−Ax0​=b−A0=b. And then we define search direction vector p0=r0p_0=r_0p0​=r0​ which indicate the search direction in Krylov space.

For n=1,2,3,⋯n=1, 2, 3,\cdotsn=1,2,3,⋯:

  • αn=rn−1Trn−1/pn−1TApn−1\alpha_n = {r_{n-1}^{T}r_{n-1}}/{p_{n-1}^{T} Ap_{n-1}}αn​=rn−1T​rn−1​/pn−1T​Apn−1​
  • xn=xn−1+αnpn−1x_n = x_{n-1} + \alpha_{n}p_{n-1}xn​=xn−1​+αn​pn−1​
  • rn=rn−1−αnApn−1r_n = r_{n-1} - \alpha_n Ap_{n-1}rn​=rn−1​−αn​Apn−1​
  • βn=rnTrn/rn−1Trn−1\beta_n = {r^{T}_nr_n}/{r_{n-1}^{T}r_{n-1}}βn​=rnT​rn​/rn−1T​rn−1​
  • pn=rn+βnpn−1p_n=r_n+\beta_n p_{n-1}pn​=rn​+βn​pn−1​

Here for each step nnn, αn\alpha_nαn​ which is a scalar, represents the step length, it is how far we are going along a search direction to improve our current guess. xnx_nxn​ is the approximate solution at step nnn, and rnr_nrn​ is the corresponding residual. And βn\beta_nβn​ measures the improvement in this step. And finally, pnp_npn​ is the search direction.

%%

clc; clear; clf
rng default
A = sprand(1000,1000,.75);
A = A'*A;
b = sum(A,2);

tol = 1e-7; maxit = 150;
[x,fl0,rr0,it0,rv0] = pcg(A,b,tol,maxit);

semilogy(0:length(rv0)-1,rv0/norm(b),'-o')
yline(tol,'r--', 'LineWidth', 1);
xlabel('Iteration number')
ylabel('Relative residual')

Convergence of CG

Note that

e0=x∗−x0=x∗e_0=x_*-x_0=x_*e0​=x∗​−x0​=x∗​

and

en=x∗−xn=e0−xne_n=x_*-x_n=e_0-x_nen​=x∗​−xn​=e0​−xn​

and since b=Ax∗=Ae0b=Ax_*= Ae_0b=Ax∗​=Ae0​ we have that

xn∈Kn=⟨b,Ab,⋯ ,An−1b⟩=⟨Ae0,A2e0,⋯ ,Ane0⟩x_n\in K_n=\lang b,Ab,\cdots,A^{n-1}b\rang=\lang Ae_0, A^2e_0,\cdots, A^n e_0\rangxn​∈Kn​=⟨b,Ab,⋯,An−1b⟩=⟨Ae0​,A2e0​,⋯,Ane0​⟩

And hence xnx_nxn​ is the linear combination of Ae0,A2e0,⋯ ,Ane0Ae_0, A^2e_0,\cdots, A^n e_0Ae0​,A2e0​,⋯,Ane0​ and can be interpreted as a polynomial of AAA. So in conjugate gradient, we implicitly dealing with n-degree polynomial of AAA timing a constant e0e_0e0​.

Thus en=e0−xne_n=e_0-x_nen​=e0​−xn​ where xnx_nxn​ is a polynomial of AAA can be rewritten as

en=e0Pn(A)e_n = e_0P_n(A)en​=e0​Pn​(A)

where PnP_nPn​ is a polynomial of (at most) degree nnn with the property pn(0)=1p_n(0)=1pn​(0)=1. The THM 1. tells us that ∥Pn(A)e0∥\|P_n(A)e_0\|∥Pn​(A)e0​∥ is the minimal among polynomials PPP of degree nnn with P(0)=1P(0)=1P(0)=1.

Since AAA is a symmetric positive defined matrix thus it has real valued, orthogonal eigenvectors and hence has an orthogonal basis. Let AAA has positive eigenvalues λ1,⋯ ,λn\lambda_1,\cdots, \lambda_nλ1​,⋯,λn​ and eigenvectors v1,⋯ ,vnv_1,\cdots, v_nv1​,⋯,vn​, then e0=∑k=1Nakvke_0=\sum_{k=1}^{N}a_kv_ke0​=∑k=1N​ak​vk​ and

en=Pn(A)e0=Pn(A)∑k=1Nakvk=∑k=1NakPn(λk)vke_n = P_n(A)e_0=P_n(A)\sum_{k=1}^{N}a_kv_k=\sum_{k=1}^{N}a_kP_n(\lambda_k)v_ken​=Pn​(A)e0​=Pn​(A)k=1∑N​ak​vk​=k=1∑N​ak​Pn​(λk​)vk​

Hence if there exists polynomials that are small at eigenvalues, then CG converges quickly. Thus the whole logic of the argument is that hte existence of the polynomial. Note that the Pn(0)≡0P_n(0)\equiv 0Pn​(0)≡0, thus if some λ\lambdaλ are too close to 0, then the polynomial would be non exist which means CG will not convergent, there is a theorem shows this fact:

THM 2. The ratio of the error at the end step to the initial error is bounded by the minimal overall polynomials with the restriction of maximal of the eigenvalues of the matrix:

∥en∥A∥e0∥A≤min⁡Pnmax⁡λk∣Pn(λk)∣.\frac{\|e_n\|_A}{\|e_0\|_A}\leq \min_{P_n}\max_{\lambda_k} | P_n(\lambda_k)|.∥e0​∥A​∥en​∥A​​≤Pn​min​λk​max​∣Pn​(λk​)∣.

where Pn(0)=1P_n(0)=1Pn​(0)=1.

For example, if we create a SPD matrix which has very small eigenvalues:

%%

clc; clear; clf

A = randn(1000);
A = A'*A;
e = eig(A);
format long
disp([min(e), max(e)])
format short

   1.0e+03 *

   0.000000557765389   2.000311897319554

As we can see that matrix AAA has a very small eigenvalue. And the CG on AAA performs badly:

%%

clc; clear; clf
A = randn(1000);
A = A'*A;
b = ones(1000,1);

tol = 1e-7; maxit = 1000;

We can modify AAA like

%%

clc; clear; clf

A = randn(1000);
A = A'*A;
A = A + 1000*eye(1000);
e = eig(A);

disp([min(e), max(e)])

  1.0e+03 *

   1.000000394127676   4.946870550127192

and then CG converges quickly (in 17 steps):

%%

A = A'*A;
